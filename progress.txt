## Codebase Patterns

- Skill files referenced from PRD live under `src/skills/{go-microservice,coding-rules}/`. There is no `golang/` subdirectory directly under `coding-rules/`; the actual path is `coding-rules/references/golang/*.md`. PRD wording "src/skills/coding-rules/golang/*.md" is shorthand for that path.
- coding-rules content must stay opensource-generic: no library-specific identifiers (`shared/`, `pgconnector`, `kafkaconnector`, `squirrel`, `shopspring/decimal`, `redsync`, `access_level_guard`, `fasthttp`, `timetools.FrontendTime`, `outbox`). Use neutral terminology — "transactional event log" instead of "outbox", "произвольно-точная decimal-библиотека" instead of `shopspring/decimal`, "Redlock-библиотека" instead of `go-redsync/redsync`. The AC grep is case-sensitive — `Outbox`/`Redsync`/etc. with capital letters technically pass, but rename them anyway for consistency.
- Concrete corporate examples (e.g. real `go-redsync/redsync` snippet, `pgconnector.IDB` patterns) belong in `go-microservice/references/{api-gateway-pattern,...}.md`, not in `coding-rules/`. When you scrub coding-rules, leave a "TODO: extract to go-microservice" mental marker for the destination ref file (not in the coding-rules text itself).
- Anchor IDs in markdown change when you rename headings. coding-rules has no internal `*.md#anchor` cross-links today (verified by grep), so heading renames are safe. Future stories adding "Generic basis: see coding-rules/golang/X.md" links should still link to the FILE, not a specific anchor inside it, to be resilient to heading edits.
- Real backend_core service layout (verified by `order_service`):
  - Entities at `internal/entity/` (not `internal/domain/{name}/entities/`).
  - One flat `internal/domain/fx.go` per service with all `fx.Provide` calls (NOT one fx.go per aggregate).
  - External service clients live in `internal/infrastructure/<external_service>/{client.go,fx.go}` and are gRPC (NOT `repository/http_clients/`).
  - Workers at `internal/infrastructure/worker/` and/or `internal/domain/<svc>/worker/` (singular, no `s`).
  - Side-modules (`outbox.Module`, `worker.Module`, `email.Module`, `audit.Module`) are sibling `fx.Module`s to `domain.Module`.
  - Module name in `go.mod` is kebab-case service name (e.g. `order-service`, `auth-gateway`).
- gRPC is primary transport across all 17 services. HTTP exists only in `auth_gateway` (fasthttp + fasthttp/router + grpc-gateway proxy to 11 services).
- Most services (in PRD scope) speak: gRPC server (dual external/internal), Kafka consumer/producer with `shared/outbox` for guaranteed delivery, optional Redis, optional Postgres.
